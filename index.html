Provenance — AI-Native Dark-Store Intake Quality Gate In Build

Java 21 · Spring Boot 3 · PostgreSQL · Redis · Kafka · AWS · Gemini 2.5 Vision · React
Problem
Curated-grocery dark stores need to pre-verify quality at intake — not assume it. A worker scanning a crate of mangoes can't wait 5s for an LLM, and a false-accept (ships rotten produce) is worse than a slow gate.
Approach
Three-path Spring Boot 3 architecture: sync intake returns 202 in <100ms, async Kafka-driven Gemini Vision verification across 5 categories × 5 quality dimensions, read-side audit. Redis caches per-SKU reference data with prompt-version-keyed invalidation.
Outcome
Sustained 100 intakes/sec at <100ms p95 under k6 load test (t3.small). Median full decision ~5s. LLM timeouts default to HOLD with a human-review flag — quality moat preserved even under degraded service.

ForensIQ — AI Credit Intelligence Engine

Python · Gemini 2.5 Vision · Groq Llama 3.3 70B · LightGBM · SHAP · NetworkX · Streamlit
Problem
Indian MSME credit underwriting takes ~2 weeks per application. Credit officers manually parse bank statements, GST returns, and legal notices — no standardised scoring, no traceability of why a verdict was reached.
Approach
4-stage appraisal service: zero-OCR Gemini Vision pipeline (9-class transaction taxonomy) feeds 5 deterministic Python tests (NetworkX cycle detection, GST–bank reconciliation, etc.) ahead of a parallel 4-dimension research layer synthesised by Groq Llama 3.3 70B.
Outcome
LightGBM default-prediction model at AUC 0.996 with SHAP per-feature attribution. Every sub-score traceable to a source document and a specific test. Hard IBC/NCLT/SARFAESI regulatory override enforces RBI prudential norms. Compresses turnaround ~2 weeks → ~3 min.
